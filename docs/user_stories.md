# User Stories

## Users
### As a student, I want to complete a quick survey so that I can receive personalized study spot recommendations.

### As a student, I want to view recommended study spots on an interactive campus map so that I can easily find them.

### As a student, I want to filter study spots by noise level so that I can choose an environment that matches my study style.

### As a student, I want to filter study spots by busyness level so that I can find a quieter space during peak hours.

### As a student, I want to mark a study spot as a favorite so that I can return to it later.

---

## Stakeholders
### As a stakeholder, I want to see a live demo of the app so that I can assess the team’s implementation of Scrum and technical design.

### As a stakeholder, I want clear documentation of sprints and user stories so that I can evaluate progress and process quality.

### As a stakeholder, I want to view each team member’s commits and issues on GitLab so that I can verify fair contribution and collaboration.

---

## Developers
### As a developer, I want to design and implement the MySQL database so that I can store and retrieve study spot data efficiently.

### As a developer, I want to connect FastAPI to the MySQL database so that data flows smoothly between backend and frontend.

### As a developer, I want to connect the React frontend to the backend API so that the app displays real-time data.

### As a developer, I want to integrate Mapbox into the frontend so that I can display study spot locations visually.

### As a developer, I want to create a responsive and user-friendly interface so that students can navigate the app easily.

### As a developer, I want to write setup and architecture documentation so that others can understand and run the project easily.

### As a developer, I want to seed the database with sample study spot data so that the system can be tested before deployment.
